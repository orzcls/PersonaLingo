# Conversation Guide — 引导对话补充细节

> 当问卷填写完成后，Agent 按本指南追问 3-6 轮，目的是让锚点故事、真实经历、词汇弱项更具体。
> 所有对话轨迹追加写入 `corpus/<corpus_id>/dialogue.md`（Markdown 原文，保留时间戳）。

## 1. 追问原则

1. **一次一问**：每轮仅提一个聚焦问题，避免信息过载。
2. **5W1H 展开**：当用户给出抽象回答时，按 When / Where / Who / What / Why / How 逐项钻取具体细节。
3. **可落地素材优先**：优先把"形容词式自述"转成"动词 + 事实"（例：`"我很热爱编程"` → `"在 ICPC 2024 区域赛上用 C++ 在 40 分钟内解出动态规划题，团队晋级"`）。
4. **禁止引导作答**：不要暗示 IELTS 官方"好答案"，保留学习者原生词汇与口音习惯。
5. **风格采样**：留意用户常用句式、口头禅、错误模式，原样抄录一到两句作为 `language_samples`。

## 2. 追问模板（按场景挑 3-6 条）

### 2.1 人物锚点
- "刚才提到的 {person_name}，最近一次让你印象深刻的对话是什么内容？尽量还原当时说的话。"
- "如果用一个词形容他/她对你的影响，会是什么？然后用一个具体事件支撑这个词。"

### 2.2 事件锚点
- "关于 {event_name}：开始之前你担心的是什么？最终结果是怎样的？过程中最难的一个决定是什么？"
- "这件事里，你做过最没有信心但还是做了的动作是什么？"

### 2.3 兴趣深化
- "你在 {interest} 上和别人聊最能点燃你的一个子话题是什么？给一个例子。"
- "你最近读过 / 看过 / 用过的 {interest} 相关内容？名字 + 一句评价。"

### 2.4 物品 / 地点
- "那件 {object} 是怎么得到的？现在放在哪里？多久会用一次？"
- "到 {place} 最近一次是什么时候？当时和谁在一起？发生了什么？"

### 2.5 词汇 / 句型薄弱点验证
- "请用 3 个你最常用的连接词连一句完整的话描述昨天。我会把它记下来做样本。"
- "描述一下此刻窗外（或身边环境），尽量用 5 个以上形容词。"

### 2.6 目标校准
- "如果下周就要考试，你最怕遇到哪类题目？说出一个具体话题（例：Describe a place you want to visit）。"

## 3. 结束条件

满足以下任一条件即结束对话，进入下一步（蒸馏）：
- 已获得 ≥ 3 个可支撑锚点故事的具体事件；
- 已采样 ≥ 5 句 `language_samples`；
- 用户主动提出"可以了"或追问超过 6 轮。

## 4. 输出产物

`corpus/<corpus_id>/dialogue.md` 结构示例：

```markdown
# Dialogue Log — corpus/<corpus_id>

## Round 1 · 2026-05-11 10:12
**Agent:** 刚才提到的 ICPC 2024 区域赛，你担心过哪一刻？
**User:** 第 90 分钟我们还是 rank 40+，我一度想弃题……

## Round 2 · 2026-05-11 10:14
...
```

结束时在文件末追加：

```markdown
---
## Summary
- anchor_events: [ICPC 2024 Regional, Harmony OS cert exam, ...]
- language_samples: ["I was like totally frozen for a second.", ...]
- weakness_evidence: ["过去式常用成一般现在时：I go there yesterday."]
```
